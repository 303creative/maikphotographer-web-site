LO QUE NECESITA MEJORA (DEBILIDADES)
================================================================================

🔴 CRÍTICO — ACCESIBILIDAD

1. SIN FOCUS-VISIBLE ESTADOS
   - Tab navigation NO muestra dónde está el cursor
   - WCAG A requirement falta
   - Solución: Agregar en CSS/main.css:
     :focus-visible { outline: 2px solid var(--accent); outline-offset: 4px; }

🟠 IMPORTANTE — UX

2. PORTFOLIO FILTROS SIN INDICADOR VISUAL
   - Botón .port-filter.active existe en JS pero NO hay CSS diferenciador
   - Usuario no ve cuál filtro está seleccionado
   - Solución: Agregar:
     .port-filter.active { color: var(--accent); border-bottom: 2px solid var(--accent); }

3. FORMULARIO SIN VALIDACIÓN VISUAL
   - Campo required pero input:invalid no tiene color roja
   - Usuario no sabe qué falló antes de submit
   - Solución: Agregar:
     input:invalid { border-color: #FF4444; background: rgba(255,68,68,0.05); }

4. MOBILE CTA BAR NO ACTIVADA
   - Elemento .mobile-cta-bar existe en HTML
   - CSS lo oculta globalmente (display: none)
   - No hay media query para mostrarlo en 375px
   - Solución: Agregar:
     @media (max-width: 768px) { .mobile-cta-bar { display: block; } }

🟡 MENOR — PERFORMANCE & FUNCIONALIDAD

5. SCROLL VIDEO MUY PESADO
   - 6.4MB puede causar lag en 3G/4G
   - Recomendación: Comprimir < 3MB o usar WebM (libvpx-vp9)

6. HOVER STATES EN GHOST BUTTONS INCOMPLETOS
   - .btn-ghost:hover solo cambia color, no transform
   - Considerar: transform: translateY(-2px)

7. STATS COUNTER NO CONFIRMADO
   - HTML tiene .about-stats con 4 items (200+, 5+, 3, 48h)
   - Main.js NO muestra CountUp animation visible
   - Verificar si es estático o si falta JavaScript
